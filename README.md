# Wisecow Application – Containerization & Kubernetes Deployment

## Overview

This project demonstrates the containerization, deployment, and automation of the Wisecow application using Docker, Kubernetes, and GitHub Actions.

Wisecow is a lightweight Bash-based web application that serves random wisdom quotes generated by `fortune` and displayed using `cowsay`. The application listens on port `4499` and returns the generated content through a simple HTTP interface.

## Project Objectives

* Containerize the Wisecow application using Docker.
* Deploy the application on Kubernetes.
* Expose the application using a Kubernetes Service.
* Implement CI/CD using GitHub Actions.
* Automate Docker image build and push operations.
* Automate Kubernetes deployments.
* Secure application access using TLS via Kubernetes Ingress.

## Dockerization

### Build Docker Image

```bash
docker build -t wisecow:v1 .
```

### Run Container

```bash
docker run -d -p 4499:4499 wisecow:v1
```

### Verify

```bash
curl http://localhost:4499
```

## Kubernetes Deployment

### Deploy Application

```bash
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
kubectl apply -f k8s/ingress.yaml
```

### Verify Resources

```bash
kubectl get pods
kubectl get svc
kubectl get ingress
```

## Service Exposure

The application is exposed internally using a Kubernetes Service.

```yaml
type: ClusterIP
```

Ingress routes external traffic to the service.

---

## TLS Configuration

A TLS certificate is configured through Kubernetes Ingress.

### Create TLS Secret

```bash
kubectl create secret tls wisecow-tls \
  --cert=tls.crt \
  --key=tls.key
```

### Verify TLS Secret

```bash
kubectl get secrets
```
